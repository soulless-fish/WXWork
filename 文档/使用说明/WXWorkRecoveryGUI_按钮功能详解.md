# 企业微信聊天恢复工具 GUI 按钮功能详解

本文档说明 `企业微信聊天恢复工具` 图形界面中各个按钮的用途、触发条件、输出结果和常见注意事项。

适用程序：

- `<PROJECT_DIR>\dist\WXWorkRecoveryGUI.exe`
- `<PROJECT_DIR>\wxwork_recovery_gui.py`

说明：

- 当前正式 GUI 已经收敛为“聊天整理 / 聊天浏览 / 运行日志”主界面。
- 本文档中较早的“运行环境 / 目标设置 / 输出文件”等表述，更多用于解释历史按钮来源和兼容逻辑。
- 现在和你最常打交道的主要是：会话列表、整理区、完整聊天内容浏览区，以及新增的 `开启实时恢复` / `停止实时恢复`。

## 一、界面总览

GUI 主要分为以下区域：

0. 菜单栏（`文件` / `聊天整理` / `帮助`）
1. 运行环境
2. 目标设置
3. 检测到的企业账号目录
4. 候选 `WXWork.exe` 进程
5. 执行操作
6. 聊天整理导出顶部信息区
7. 左侧会话列表
8. 右侧会话信息卡与消息预览
9. 恢复日志
10. 输出文件

程序主要有两条工作流：

1. 聊天恢复工作流
   先选择企业账号ID，再扫描进程，再执行恢复。
2. 聊天整理工作流
   先恢复出 sqlite，再扫描会话，再按会话类型筛选并整理导出。

## 二、按钮总表

当前 GUI 中的可点击按钮如下：

1. 运行环境
   - `重新加载`（恢复引擎）
   - `打开`（企业微信目录）
   - `重新加载`（整理引擎）
2. 目标设置
   - `刷新账号列表`
   - `清空 PID`
   - `浏览`
   - `打开`
3. 候选 WXWork 进程
   - `扫描匹配进程`
   - `使用所选 PID`
   - `打开企业微信目录`
4. 执行操作
   - `开始恢复`
   - `刷新结果`
   - `打开最新 CSV`
   - `打开最新报告`
   - `打开操作文档`
   - `打开交接文档`
5. 聊天整理导出
   - `打开目录`
   - `打开最近导出索引`
   - `打开全部会话总表`
   - `打开最近整理时间线`
   - `开启实时恢复`
   - `停止实时恢复`
   - `刷新会话列表`
   - `刷新当前预览`
   - `整理所选会话`
   - `整理当前筛选结果`
6. 输出文件
   - `打开所选文件`
   - `打开输出目录`
   - `刷新`

另外还有 4 个双击行为：

1. 双击“检测到的企业账号目录”中的一行
   自动选中该企业账号ID，并立即开始扫描匹配进程。
2. 双击“候选 WXWork 进程”中的一行
   等价于点击 `使用所选 PID`。
3. 单击“聊天整理导出”中的会话列表一行
   在右侧加载该会话的消息预览、发送人、发送时间和命名依据。
4. 双击“聊天整理导出”中的会话列表一行
   等价于点击 `整理所选会话`。
5. 双击“输出文件”中的文件一行
   等价于点击 `打开所选文件`。

## 三、运行环境区域

### 1. `重新加载`（恢复引擎）

位置：

- `运行环境` -> `恢复引擎`

作用：

- 重新加载聊天恢复核心脚本 `recover_wxwork_partial_messages.py`
- 用于排查“恢复引擎加载失败”或脚本刚更新后的重新读取

适合什么时候点：

- 你刚刚更新了恢复脚本
- GUI 日志里出现恢复引擎加载失败
- 想确认 GUI 当前读取的是最新脚本

执行结果：

- 成功时：状态显示 `已就绪`
- 失败时：状态显示 `加载失败：...`

### 2. `打开`（企业微信目录）

位置：

- `运行环境` -> `企业微信目录`

作用：

- 打开本机企业微信数据目录
- 通常是 `C:\Users\<用户名>\Documents\WXWork`

适合什么时候点：

- 需要核对本机有哪些企业账号目录
- 想确认哪一个数字目录对应当前账号

### 3. `重新加载`（整理引擎）

位置：

- `运行环境` -> `整理引擎`

作用：

- 重新加载聊天整理脚本 `organize_wxwork_recovered_messages.py`
- 用于排查“整理引擎加载失败”或整理脚本更新后的重新读取

执行结果：

- 成功时：状态显示 `已就绪`
- 失败时：状态显示 `加载失败：...`

## 四、目标设置区域

### 1. `刷新账号列表`

位置：

- `目标设置`

作用：

- 扫描企业微信文档目录下的数字目录
- 刷新左侧“检测到的企业账号目录”表格

输出内容：

- 企业账号ID
- 是否存在 `message.db`
- 目录修改时间

适合什么时候点：

- 刚切换账号后
- 新增账号后
- 目录显示不完整时

### 2. `清空 PID`

位置：

- `目标设置`

作用：

- 清空手工锁定的 PID

适合什么时候点：

- 之前锁定了某个 PID，但现在想重新自动扫描
- 怀疑之前选错进程

说明：

- 清空后，恢复流程会重新根据企业账号ID自动寻找最匹配的 `WXWork.exe`

### 3. `浏览`

位置：

- `目标设置` -> `输出目录`

作用：

- 选择恢复输出目录

默认目录：

- `<PROJECT_DIR>`

会影响什么：

- 聊天恢复输出文件位置
- 聊天整理导出默认会跟随到这个目录下的 `organized_external_groups`

### 4. `打开`（输出目录）

位置：

- `目标设置` -> `输出目录`

作用：

- 打开当前设置的恢复输出目录

适合什么时候点：

- 想直接查看恢复产物
- 想确认输出目录是否正确

## 五、检测到的企业账号目录区域

这个区域本身没有按钮，但有两个重要交互：

### 1. 单击一行

作用：

- 把该行的企业账号ID填入“目标设置”里的 `企业账号ID`

### 2. 双击一行

作用：

- 先选中该企业账号ID
- 再立即执行 `扫描匹配进程`

适合什么时候用：

- 你已经从目录列表里确认了正确账号，想少点一步

## 六、候选 WXWork 进程区域

## 七、新增的实时恢复按钮

### 1. `开启实时恢复`

位置：

- `聊天整理导出` -> `实时恢复最新消息`

作用：

- 按你填写的秒数间隔自动重跑一次聊天恢复
- 每轮成功后自动刷新：
  - 最新 sqlite
  - 会话列表
  - 当前选中的聊天内容预览

适合什么时候点：

- 你正在盯某个会话，想持续看到最新恢复结果
- 不想每隔十几秒就手工点一次 `开始恢复`

注意事项：

- 这不是直接监听企业微信新消息事件，而是自动定时重跑恢复。
- 企业微信必须保持打开状态。
- 最好先打开目标聊天窗口；如果还要补更多历史，先上滑消息再开启。
- 当前已经允许设置为 `1` 秒。
- 如果单轮恢复本身超过 `1` 秒，GUI 会在上一轮结束后立即补跑下一轮，而不会并发启动多轮恢复。

### 2. `停止实时恢复`

位置：

- `聊天整理导出` -> `实时恢复最新消息`

作用：

- 停止后续自动恢复轮次

执行结果：

- 如果当前没有任务在跑，会立即停止。
- 如果当前正好有一轮恢复在执行，会在这一轮结束后停止，不再继续下一轮。

### 1. `扫描匹配进程`

位置：

- `候选 WXWork 进程`

作用：

- 根据当前输入的企业账号ID
- 扫描所有 `WXWork.exe`
- 查找哪个进程内存里包含 `<企业账号ID>\Data\message.db`

前置条件：

- 已输入或选中正确的企业账号ID
- 企业微信目标账号处于打开状态

执行结果：

- 右侧列表中出现候选 `WXWork.exe` 进程
- 列包括 PID、是否 WOW64、路径命中数、示例命中地址

说明：

- 路径命中数越高，通常越可能是正确进程
- 如果扫描不到，通常说明账号没开、账号选错、或企业微信没把目标账号载入内存

### 2. `使用所选 PID`

位置：

- `候选 WXWork 进程`

作用：

- 把当前选中的候选进程 PID 锁定到“目标设置”的 `可选 PID`

适合什么时候点：

- 你已经从候选列表里确认了某个 PID
- 你不想让恢复流程重新自动选择进程

说明：

- 锁定 PID 后，恢复时优先使用这个进程
- 如果进程已退出，恢复会失败

### 3. `打开企业微信目录`

位置：

- `候选 WXWork 进程`

作用：

- 再次打开企业微信数据目录

用途：

- 快速回看账号目录
- 检查企业账号ID是否对应正确

### 4. 双击进程行

作用：

- 等价于点击 `使用所选 PID`

## 七、执行操作区域

### 1. `开始恢复`

位置：

- `执行操作`

作用：

- 启动聊天恢复主流程
- 从目标 `WXWork.exe` 内存中提取已解密的 SQLite page cache
- 解析 `message_table`
- 导出恢复结果

前置条件：

1. 企业微信已打开目标账号
2. 最好已打开目标聊天并上滑更多历史
3. 已填写企业账号ID
4. 如有需要，可先锁定 PID

主要输出：

- `wxwork_<企业账号ID>_partial_<PID>.json`
- `wxwork_<企业账号ID>_partial_<PID>.sqlite`
- `wxwork_<企业账号ID>_partial_<PID>_readable.csv`
- `wxwork_<企业账号ID>_partial_<PID>_report.md`

说明：

- 这是“部分恢复”，不是整库解密
- 想恢复更多消息，先去企业微信里继续上滑聊天，再重跑

### 2. `刷新结果`

位置：

- `执行操作`

作用：

- 重新扫描当前恢复输出目录
- 刷新“输出文件”列表

适合什么时候点：

- 刚完成恢复后
- 手工拷贝了结果文件后
- 想确认目录里最新文件是否已出现

### 3. `打开最新 CSV`

位置：

- `执行操作`

作用：

- 打开当前输出目录里最新的 `*_readable.csv`

适合什么时候点：

- 想最快查看最近一次恢复的可读消息结果

### 4. `打开最新报告`

位置：

- `执行操作`

作用：

- 打开当前输出目录里最新的 `*_report.md`

适合什么时候点：

- 想快速看恢复摘要、热门会话、最近消息

### 5. `打开操作文档`

位置：

- `执行操作`

作用：

- 打开恢复方法说明文档

对应文件：

- `<PROJECT_DIR>\文档\交接资料\WXWork_Chat_Recovery_Playbook.md`

### 6. `打开交接文档`

位置：

- `执行操作`

作用：

- 打开给另一台机器或另一个 Codex 使用的交接文档

对应文件：

- `<PROJECT_DIR>\文档\交接资料\WXWork_Recovery_Codex_Handoff.md`

## 八、聊天整理导出区域

这个区域用于把恢复出来的聊天，按“一个会话一个目录”的方式整理成更容易阅读的文件。

### 1. `打开目录`

位置：

- `聊天整理导出` -> `整理输出`

作用：

- 打开整理后的输出目录

默认目录：

- `<恢复输出目录>\organized_external_groups`

典型内容：

- 每个会话一个子目录
- 各类 `*索引.csv`
- `全部会话总表.csv`

### 2. `打开最近导出索引`

位置：

- `聊天整理导出` -> `最新 sqlite`

作用：

- 打开最近一次聊天整理生成的筛选结果索引文件

对应文件：

- 可能是：
  - `外部群索引.csv`
  - `内部群索引.csv`
  - `单聊索引.csv`
  - `助手应用索引.csv`
  - `当前筛选会话索引.csv`

注意：

- 这类索引 CSV 只是“目录/摘要表”，不是单个会话的逐条聊天记录。
- 真正的聊天内容要进会话目录里看 `时间线.csv`。

如果提示“未找到文件”：

- 说明你还没有执行过聊天整理

### 3. `刷新会话列表`

位置：

- `聊天整理导出`

作用：

- 读取最新恢复 sqlite
- 扫描其中全部已恢复会话
- 在下方表格中列出每个会话
- 默认会先显示“聊天会话（内/外部）”，也就是外部群 `R:` 加内部/部门群 `R:`
- 可通过“显示范围”继续切换到全部会话，或筛选 `S:`、`Y:`、`F:` 等其它类型

前置条件：

- 当前恢复输出目录中已经有恢复出的 sqlite 文件

会话列表列信息：

- 会话名称
- 对方ID
- 会话ID
- 会话类型
- 消息数
- 发送人数
- 最后时间

### 4. `整理所选会话`

位置：

- `聊天整理导出`

作用：

- 只整理当前选中的一个会话

前置条件：

1. 先点击 `刷新会话列表`
2. 在会话表格中选中一个会话

输出结果：

- 对应会话目录下生成：
  - `时间线.csv`
  - `会话时间线.md`
  - `发送人映射.csv`
  - `媒体索引.csv`
  - `会话摘要.json`
  - `导出文件说明.txt`

说明：

- `时间线.csv` 是该会话的全量消息明细
- `发送人映射.csv` 中的 `sender_id` 是当前最稳定的恢复标识，`sender_name_guess` 只是根据消息内容推测出的名字
- `媒体索引.csv` 只保留带媒体线索的消息
- `R:` 前缀不一定都是真正的外部群；当前 GUI 已尽量把明显的内部/部门群拆分出去
- 单聊会优先尝试读取本机 `Index\data_index.db` 的联系人索引，能补回一部分真实联系人名/备注名
- 如果单聊拿不到可信名字，会退回 `单聊_<对方ID>`，并在表格里单独显示 `对方ID`
- 外部群真实群名未必都在当前恢复消息里，但如果本机索引里仍有会话名，脚本会优先拿来补全
- 会话名称不是 100% 等于联系人正式名称或真实群名，因此应同时参考 `对方ID` 和 `会话ID`

### 5. `整理当前筛选结果`

位置：

- `聊天整理导出`

作用：

- 一次性整理当前筛选条件下看到的全部会话
- 如果筛选是“聊天会话（内/外部）”，就会同时整理当前恢复结果里的外部群和内部/部门群
- 如果筛选是“外部群 (R:)”，就会整理当前恢复结果中的全部外部群
- 如果筛选是“内部群/部门群 (R:)”，就会整理被识别为内部通知群、部门群的 `R:` 会话
- 如果筛选是“助手/应用 (Y:)”，就会整理全部 `Y:` 会话

输出结果：

- 每个会话生成一个独立目录
- 同时生成总索引：
  - 一个与当前筛选类型对应的索引文件
  - 一个 `全部会话总表.csv`

适合什么时候点：

- 你想一次看当前某一类会话
- 需要批量导出给别人查看

### 6. 双击会话行

作用：

- 等价于点击 `整理所选会话`

### 7. `打开最近整理时间线`

位置：

- `聊天整理导出` -> `最新 sqlite`

作用：

- 打开最近一次整理生成的聊天时间线文件
- 优先用于查看真正的聊天明细，而不是索引表

说明：

- 如果是整理单个会话，GUI 现在会尽量直接打开该会话的 `时间线.csv`
- 如果你手动点这个按钮，也会优先打开最近生成的 `时间线.csv`

## 十、按钮使用顺序建议

### 场景 A：恢复聊天记录

推荐顺序：

1. `刷新账号列表`
2. 在企业账号目录里选中正确账号
3. `扫描匹配进程`
4. 可选：`使用所选 PID`
5. `开始恢复`
6. `打开最新 CSV`
7. `打开最新报告`

### 场景 B：把恢复结果按会话整理

推荐顺序：

1. 先完成一次聊天恢复
2. `刷新会话列表`
3. 视需要切换“显示范围”
4. 选中一个会话
5. `整理所选会话`
6. `打开最近导出索引`
7. `打开目录`

### 场景 C：批量整理某一类会话

推荐顺序：

1. 先完成一次聊天恢复
2. `刷新会话列表`
3. 先在“显示范围”中选好目标类型
4. `整理当前筛选结果`
5. `打开最近导出索引`

## 十一、常见提示说明

### 1. “缺少企业账号ID”

含义：

- 还没有输入或选择企业账号ID

处理：

- 从目录列表选一个，或者手工输入那串数字

### 2. “企业账号ID无效”

含义：

- 输入的企业账号ID不是纯数字

处理：

- 只输入数字目录名，不要带空格或其他字符

### 3. “未选择进程”

含义：

- 点了 `使用所选 PID`，但没有选中候选进程

### 4. “未选择会话”

含义：

- 点了 `整理所选会话`，但没有选中会话列表中的某一行

### 5. “任务进行中”

含义：

- 当前已有恢复、扫描或整理任务在后台运行

处理：

- 等当前任务结束后再点新的核心执行按钮

### 6. “未找到文件”

可能原因：

1. 还没有生成相应输出
2. 输出目录切换了
3. 你要打开的索引或报告尚未产生

## 十二、输出文件含义

### 恢复阶段输出

- `*.json`
  保存恢复元数据和原始行内容
- `*.sqlite`
  保存恢复出的 `message_table_partial`
- `*_readable.csv`
  适合 Excel 查看
- `*_report.md`
  适合快速阅读

### 整理阶段输出

- `全部会话总表.csv`
  本次扫描到的全部会话总表，便于判断到底恢复到了哪些类型
- `外部群索引.csv`、`内部群索引.csv`、`单聊索引.csv`、`助手应用索引.csv`、`当前筛选会话索引.csv`
  本次导出的筛选结果索引，具体名称取决于当前筛选范围
- `<会话目录>\时间线.csv`
  单个会话的全量消息明细表
- `<会话目录>\会话时间线.md`
  单个会话的可读时间线
- `<会话目录>\发送人映射.csv`
  发送人映射和姓名推测
- `<会话目录>\媒体索引.csv`
  只保留带媒体线索的消息
- `<会话目录>\会话摘要.json`
  单个会话的摘要
- `<会话目录>\导出文件说明.txt`
  当前目录文件用途说明
